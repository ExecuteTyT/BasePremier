## 2. РАБОЧИЙ ПРОТОКОЛ

Это правила работы на весь проект. Они важнее любого тикета.

1. **Один тикет за раз.** Не комбинируй BS-02 и BS-03 в одной сессии «потому что они похожи». Каждый тикет — отдельный коммит.

2. **Перед написанием кода — план в чате.** Покажи мне:
   - Какие файлы создашь / изменишь (со списком путей)
   - Какие npm-пакеты установишь (если есть)
   - Какой результат я увижу в браузере / терминале
   - Acceptance criteria из ROADMAP.md, которые закроешь
   
   Жди моего «ок, поехали» прежде чем писать.

3. **TodoWrite — обязательно** для тикетов из 3+ шагов. Помечай каждый шаг done сразу после завершения.

4. **`pnpm typecheck && pnpm lint` перед каждым commit.** Без исключений. Если что-то ломается — сначала фиксим, потом коммитим.

5. **Используй субагентов из `.claude/agents/`** для специализированных задач:
   - Дизайн-токены / шрифты → `design-importer`
   - GSAP / Lenis / page transitions / 3D-сцены → `animation-engineer`
   - Тексты, статьи, microcopy, OG → `copywriter`
   - JSON-LD / sitemap / robots / meta → `seo-auditor`
   - Lighthouse / Core Web Vitals → `perf-auditor`
   - WCAG 2.2 AA / axe-core → `a11y-auditor`
   - Sanity схемы / GROQ-запросы → `content-modeler`
   
   Не пытайся всё делать сам — субагенты заточены под их задачи.

6. **Если что-то противоречит:** `docs/*.md` побеждает `CLAUDE.md`. `BRAND.md` побеждает любые «творческие» порывы. Никогда не выдумывай токены, цвета или шрифты — бери из `BRAND.md`.

7. **Стоп-сигналы — спрашивай меня:**
   - В тикете указан внешний ресурс / API ключ / GPS-координаты, которых у меня ещё нет
   - Acceptance criteria неоднозначны (можно решить двумя способами)
   - Performance budget близок к пробитию (LCP > 1.6s, INP > 90ms)
   - Я не упомянул что-то, что нужно для тикета (например, фото мастера для UI-06)
   - Тикет блокирует следующий, а у меня нет ответа на зависимость
   
   В этих случаях — **остановись и задай вопрос**. Один точный вопрос экономит 30 минут реализации, которую я попрошу переделать.

8. **Коммиты по conventional-commits:**
   ```
   feat(BS-01): bootstrap Next.js 15 + TypeScript strict
   fix(HOM-01): hero LCP 2.4s → 1.5s через preload poster.avif
   docs(BRAND): update typography scale per BRAND.md §2.3
   chore(deps): upgrade @react-three/fiber to 9.0.0
   ```
   Тикет в скобках обязателен — это нить через всю историю.

9. **Документация догоняет код, а не наоборот.** Если ты изменил что-то в архитектуре или брендинге, и это противоречит `docs/*.md` — обнови документ в том же тикете. Документация, отставшая от кода, хуже её отсутствия.

10. **Performance budget — закон, не ориентир:**
    - LCP < 1.8s (target < 1.5s)
    - INP < 100ms (target < 80ms)
    - CLS < 0.05
    - JS bundle initial < 200kb gzipped
    - Lighthouse Perf ≥ 92, A11y ≥ 95, SEO = 100
    
    Если тикет не вписывается — обсудим, что урезать. Но не ломаем бюджет ради «красивее».
