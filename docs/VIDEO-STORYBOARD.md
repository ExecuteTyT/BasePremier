# VIDEO-STORYBOARD — BASE Premier

> План работы с видео-контентом на сайте. У заказчика есть готовый материал hero-видео и съёмки процесса. Документ описывает: (а) как использовать существующее, (б) какие технические требования, (в) план B на пере-съёмку, если качество существующего недостаточно.

**Версия:** 1.0 · **Дата:** 26 апреля 2026 г. · **Связано с:** [DESIGN-BRIEF.md](DESIGN-BRIEF.md), [BRAND.md](BRAND.md), [ARCHITECTURE.md](ARCHITECTURE.md)

## 0. Принципы видео в премиум-сегменте

Семь правил. Если хоть одно нарушено — видео не идёт на сайт.

1. **Без речи.** Никогда не используем голос, диктора, лозунги. Атмосфера передаётся жестом, рукой, инструментом, светом.
2. **Без музыки на старте.** Видео по умолчанию muted (требование браузеров для автоплея). Звук — только если пользователь сам включил тумблер в шапке.
3. **Длинные планы.** Один план — минимум 3 секунды. Никаких быстрых нарезок «MTV-style». Ритм медленный, как у Tom Ford или Aesop.
4. **Цветокор Navy + warm skin.** Тени уведены в Navy `#1B2A4E`, кожа — тёплая, естественная. Никакого «синего фильтра поверх всего».
5. **Минимум фокуса на лицах клиентов.** Снимаем затылки, руки, инструменты, детали интерьера. Лица клиентов — только если есть письменное согласие; для безопасности проще без них.
6. **Один эффект — одна цель.** Slow-mo используется один раз за весь ролик и только на ключевом моменте (например, момент ножниц у уха). Не везде.
7. **Без водяных знаков, без лого «BASE Premier» поверх кадра.** Бренд — в типографике страницы и в эстетике, не в watermark'е.

## 1. Где видео используется на сайте

### 1.1 Hero-секция главной страницы (`/`)

**Цель:** атмосферный hero на фоне 3D-монограммы. Здесь видео — НЕ главное (главное — 3D BP), но является вторым визуальным слоем.

**Решение по компоновке:**
- На главной hero **видео НЕ используется** на фоне. Вместо этого — 3D-монограмма BP с тёмным фоном `bg-bg-primary`.
- Это сделано осознанно: 3D + видео одновременно = двойная нагрузка на GPU + CPU, риск падения FPS на mid-range мобильных.
- **Альтернатива (план B на если 3D окажется неподъёмным):** 3D заменяется на статичный SVG poster, а на фон добавляется лёгкое атмосферное hero-видео (см. § 2).

### 1.2 Section Process (главная, секция «Как проходит визит»)

**Цель:** на каждом шаге (01–04) показывать видео-фрагмент 4–8 секунд, демонстрирующий этот этап визита.

**Шаги:**
- **01 Консультация** — кадр: рукопожатие или жест мастера к креслу, общий план зала. ~5s.
- **02 Стрижка** — крупный план ножниц/машинки, рук мастера. ~6s.
- **03 Бритьё** — тёплое полотенце на лице, опасная бритва. ~7s.
- **04 Финиш** — укладка, аромат (распыление в воздух — light beam в кадре). ~5s.

**Поведение:**
- Видео автозапускается при попадании активного шага в viewport (через ScrollTrigger).
- Loop, muted.
- Crossfade 0.4s между шагами.

### 1.3 Section Interior Gallery (галерея)

**Цель:** возможно одно из 6–8 «фото» на самом деле короткий видео-loop (например, мастер за работой издалека, или огонь в лампе).

**Решение:** на старте — все статичные фото. Если есть подходящие 2–3-секундные lopы из существующего материала — заменить 1–2 ячейки галереи. Не больше двух (иначе перебор).

### 1.4 Карточка услуги (`/services/[slug]`) — опционально

**Цель:** в верхней части страницы услуги — короткое видео процесса именно этой услуги.

**Решение:** на старте — статичная фотография. Видео добавляем фазой 2 (после launch), если есть бюджет на дополнительную съёмку.

### 1.5 Профиль мастера (`/barbers/[slug]`) — опционально

**Цель:** короткий ролик «мастер за работой» вместо или дополнительно к фото.

**Решение:** на старте — только фотопортрет (см. CONTENT-BRIEF.md). Видео — будущая итерация.

## 2. Использование существующего материала (план A)

Заказчик подтвердил: есть hero-видео и материал процессов. Нужно:

### 2.1 Аудит существующего видео (week 1)

**Что надо получить от Айрата:**
- Все исходники (или ссылки на облако).
- Указание на «лучшие» по мнению заказчика моменты.

**Что Claude/директор делает:**
- Просматривает, разделяет на логические чанки (стрижка / бритьё / интерьер общие планы / детали инструментов / процесс).
- Оценивает качество по чек-листу:

```
Чеклист качества видео-материала
─────────────────────────────────
[ ] Разрешение ≥ 1080p (1920×1080)
[ ] Стабилизация (без тряски рук)
[ ] Освещение тёплое или нейтральное (не жёлтое-флуоресцентное)
[ ] Композиция: главный объект в кадре, не обрезан
[ ] Длительность одного плана ≥ 3 секунд
[ ] Нет случайных людей в кадре (или их можно вырезать)
[ ] Без watermark / логотипов сторонних
[ ] Звук опционально, не критичен (всё равно muted)
─────────────────────────────────
```

**Решение по итогу аудита:**
- ≥ 70% планов проходят чек-лист → используем существующее, лёгкая постобработка (план A, см. § 3).
- < 70% → переснимаем (план B, см. § 4).

### 2.2 Постобработка существующего материала

**Программы:**
- Резка/нарезка: DaVinci Resolve (бесплатный) или Final Cut Pro (если у заказчика macOS) или CapCut Pro для простых задач.
- Цветокоррекция: DaVinci Resolve (профессиональный colorgrade), Lumetri в Premiere Pro.
- Сжатие: HandBrake (бесплатный) или ffmpeg в командной строке.

**Шаги постобработки:**

1. **Импорт** в монтажный стол (proxy для быстрой работы, исходники для финального экспорта).
2. **Отбор кадров** — выбираем только лучшие 30–40 секунд из всего материала на каждый шаг.
3. **Цветокоррекция:**
   - Контраст: чуть приглушаем средние тона (lift +5%, shadows down -10%).
   - Saturation: −15% общая, кожа — natural.
   - Тени уведены в Navy: в DaVinci используем Color Wheels: Lift → push slightly toward blue (Navy #1B2A4E), Gain → neutral.
   - LUT: создаём собственный LUT на основе референсов Tom Ford/Aesop, применяем + ручная корректировка по каждому клипу.
4. **Стабилизация** (если нужна): Resolve / Premiere built-in stabilizer.
5. **Финальный рендер:**
   - Master copy: ProRes 422 HQ или ProRes Proxy для архива.
   - Web-варианты: см. § 3 ниже.

### 2.3 Кадрирование под разные форматы

Каждый клип нужен в нескольких aspect-ratio:

| Использование          | Aspect ratio | Разрешение         |
|------------------------|--------------|---------------------|
| Hero-fallback (если)   | 16:9         | 1920×1080 + 3840×2160 (4K для retina) |
| Process-секция деск.   | 4:5          | 1080×1350           |
| Process-секция мобайл  | 9:16         | 1080×1920 (вертикаль) |
| Interior gallery loop  | 1:1 или 4:5  | 1080×1080 / 1080×1350 |

**Принцип:** снимаем в 4K 16:9, в монтаже кадрируем под нужное соотношение. На главной странице используем landscape, на мобильных — portrait версии.

## 3. Технические требования к видео (под Next.js + web)

### 3.1 Форматы и кодеки

**Двухформатная стратегия (modern + fallback):**

```
Каждое видео экспортируется в 2-3 формата:
─────────────────────────────────
1. WebM / VP9         — основной для современных браузеров
2. MP4 / H.264        — fallback для старых браузеров (Safari < 14)
3. AV1 (опционально)  — экспериментально для Chrome/Edge, ~30% меньше WebM
─────────────────────────────────
```

**ffmpeg команды:**

```bash
# WebM VP9 (основной)
ffmpeg -i input.mov \
  -c:v libvpx-vp9 -b:v 0 -crf 32 \
  -pass 1 -an -f null /dev/null && \
ffmpeg -i input.mov \
  -c:v libvpx-vp9 -b:v 0 -crf 32 \
  -pass 2 -an -tile-columns 6 -frame-parallel 1 \
  -row-mt 1 -movflags +faststart \
  output.webm

# MP4 H.264 (fallback)
ffmpeg -i input.mov \
  -c:v libx264 -crf 24 -preset slow \
  -profile:v high -level 4.1 \
  -pix_fmt yuv420p -movflags +faststart \
  -an output.mp4
```

**Замечания:**
- `-crf 32` для WebM ≈ `-crf 24` для H.264 по визуальному качеству.
- `-an` убирает звуковую дорожку (на сайте все видео muted).
- `-movflags +faststart` помещает meta в начало файла, чтобы страница начинала рендерить видео раньше.

### 3.2 Бюджеты по размеру файла

| Использование          | Длительность | Размер цель WebM | Размер цель MP4 |
|------------------------|--------------|-------------------|-------------------|
| Hero-fallback          | 12–15s loop  | ≤ 1.5 MB          | ≤ 2.0 MB          |
| Process-шаг            | 5–8s loop    | ≤ 800 KB          | ≤ 1.0 MB          |
| Interior loop          | 3–5s loop    | ≤ 500 KB          | ≤ 700 KB          |
| Карточка услуги (фаза 2) | 8–12s loop | ≤ 1.0 MB          | ≤ 1.3 MB          |
| Профиль мастера (фаза 2) | 10–15s loop | ≤ 1.2 MB         | ≤ 1.5 MB          |

**Если не укладываемся в бюджет:**
- Снижаем разрешение (1920→1280 или ещё ниже).
- Уменьшаем bitrate (CRF +4).
- Сокращаем длительность.
- Если всё равно не лезет — фрагмент не используется.

### 3.3 Poster (превью)

Каждое видео имеет poster image:
- Формат: AVIF (основной) + WebP (fallback).
- Разрешение: то же, что у видео (1920×1080 для hero, 1080×1350 для portrait).
- Бюджет: ≤ 80 KB AVIF, ≤ 120 KB WebP.
- Берётся из самого первого кадра видео или из «лучшего» кадра внутри ролика.

**Генерация ffmpeg:**

```bash
# Извлечь кадр на 0.5s
ffmpeg -i input.mp4 -ss 00:00:00.500 -vframes 1 -q:v 2 poster.jpg

# Сконвертировать в AVIF (через avifenc)
avifenc -q 50 -s 6 poster.jpg poster.avif

# В WebP через ffmpeg
ffmpeg -i poster.jpg -lossless 0 -q:v 80 poster.webp
```

### 3.4 Storage и доставка

**Где хранить:**
- В репозитории — НЕТ. Бинарные файлы засоряют git.
- В Sanity (CMS) — ДА. Sanity предоставляет CDN с автоматическим resizing/transformation для изображений, для видео — прямая ссылка.
- Альтернатива: прямой upload в `/public/videos/` (только для статичного hero, который не меняется). Размер папки `public/` контролируется CI-чеком (`< 5MB total videos`).

**Доставка:**
- Через Sanity CDN: URLs вида `https://cdn.sanity.io/files/{projectId}/{dataset}/{filename}.webm`.
- HTTP/2 + Brotli автоматически применяется CDN.
- Cache-Control: `public, max-age=31536000, immutable` (Sanity делает автоматически на content-hash).

## 4. Компонент `<HeroVideo />` (Next.js)

```tsx
// src/components/media/HeroVideo.tsx
'use client'

import { useEffect, useRef, useState } from 'react'

type HeroVideoProps = {
  webmSrc: string
  mp4Src: string
  posterSrc: string
  posterFallbackSrc?: string  // WebP fallback
  alt: string
  className?: string
  loop?: boolean
  autoPlay?: boolean
  priority?: boolean  // если true — eagerly loads, иначе lazy через IntersectionObserver
}

export function HeroVideo({
  webmSrc,
  mp4Src,
  posterSrc,
  posterFallbackSrc,
  alt,
  className,
  loop = true,
  autoPlay = true,
  priority = false,
}: HeroVideoProps) {
  const videoRef = useRef<HTMLVideoElement>(null)
  const [shouldLoad, setShouldLoad] = useState(priority)
  const [reducedMotion, setReducedMotion] = useState(false)

  // Detect prefers-reduced-motion
  useEffect(() => {
    const mq = window.matchMedia('(prefers-reduced-motion: reduce)')
    setReducedMotion(mq.matches)
    const handler = (e: MediaQueryListEvent) => setReducedMotion(e.matches)
    mq.addEventListener('change', handler)
    return () => mq.removeEventListener('change', handler)
  }, [])

  // Lazy-load via IntersectionObserver if not priority
  useEffect(() => {
    if (priority) return
    const node = videoRef.current
    if (!node) return

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setShouldLoad(true)
          observer.disconnect()
        }
      },
      { rootMargin: '200px' }
    )
    observer.observe(node)
    return () => observer.disconnect()
  }, [priority])

  // If reduced-motion, show only poster
  if (reducedMotion) {
    return (
      <picture className={className}>
        <source srcSet={posterSrc} type="image/avif" />
        {posterFallbackSrc && <source srcSet={posterFallbackSrc} type="image/webp" />}
        <img
          src={posterFallbackSrc || posterSrc}
          alt={alt}
          className="w-full h-full object-cover"
          loading={priority ? 'eager' : 'lazy'}
        />
      </picture>
    )
  }

  return (
    <video
      ref={videoRef}
      className={className}
      autoPlay={autoPlay && shouldLoad}
      loop={loop}
      muted
      playsInline
      poster={posterSrc}
      preload={priority ? 'auto' : 'metadata'}
      aria-label={alt}
    >
      {shouldLoad && (
        <>
          <source src={webmSrc} type="video/webm" />
          <source src={mp4Src} type="video/mp4" />
        </>
      )}
      {/* Fallback для совсем старых браузеров */}
      <img src={posterSrc} alt={alt} />
    </video>
  )
}
```

## 5. План B — пере-съёмка (если существующий материал не проходит)

Если по итогу аудита (§ 2.1) качество существующего видео недостаточно — нужна пере-съёмка. Ниже cinematic раскадровка для съёмочной смены.

### 5.1 Объём съёмки

**Что снимаем:**
- 1 hero loop (12–15s, общий план зала с движением камеры)
- 4 process-loop (по одному на каждый шаг визита, 5–8s каждый)
- 2–3 interior loop (детали интерьера, 3–5s каждый)

**Итого:** ~40–60 секунд готового материала. На это достаточно одной съёмочной смены 6–8 часов в самом салоне.

### 5.2 Команда (минимум)

- Оператор с DSLR/mirrorless (Sony A7 III/IV, Canon R5/R6, GH6 — что-то из премиум/среднего сегмента, обязательно 4K 24fps).
- Ассистент (свет / фокус-puller).
- Гаффер (свет, если оператор работает один — гаффер опционален, но желателен).
- Барбер «модель» — один из мастеров, готовый дублироваться по сценам.
- Клиент-модель — друг/коллега Айрата, согласный на лицо в кадре (письменное согласие!).
- Стилист (если есть бюджет) — следит за внешним видом «модели».

**Бюджетная альтернатива (DIY):** 
- Айрат + один оператор-фрилансер из Казани.
- Стоимость: ~30–60 тыс. ₽ за смену (оператор + базовое освещение + монтаж).

### 5.3 Сценарий съёмки

#### Сцена 1 — Hero loop (12–15s)

- **План:** общий план зала. Камера движется slowly (slider/gimbal) слева направо или dolly-in.
- **Свет:** мягкий, тёмные тени, точечная подсветка отдельных кресел или ламп.
- **Движение:** мастер вдалеке за работой, едва заметные руки, никаких лиц.
- **Идея:** «Помещение живое и спокойное, никаких суеты.»
- **Длительность:** 12–15 секунд, замкнутый по началу-концу для бесшовного loop.
- **Аудио:** не пишем (всё равно muted).

```
Кадр 1 (0–4s)   Камера двигается из дверного проёма внутрь зала.
                Видны кожаные кресла слева, бар-стойка справа.
                Низкий свет, тёплые лампы.

Кадр 2 (4–9s)   Slow zoom в направлении кресла 3 (вглубь зала).
                Силуэт мастера (со спины) работает над клиентом.

Кадр 3 (9–13s)  Камера слегка поднимается (jib up или slider up).
                Видна полка с инструментами и продукцией.

Кадр 4 (13–15s) Финальный зум на детали (кран, бритва, флакон).
                Loop возвращается к Кадру 1.
```

#### Сцена 2 — Step 01 «Консультация» (5–6s)

- **План:** средний план. Мастер и клиент друг напротив друга. Жест рукопожатия.
- **Идея:** «Уважение к запросу, диалог.»
- **Особенности:** только руки и плечи, лица не нужны.

#### Сцена 3 — Step 02 «Стрижка» (6–8s)

- **План:** макро. Ножницы или машинка вблизи затылка/виска.
- **Идея:** «Точность, инструмент в работе.»
- **Slow-motion:** опциональный момент — ножницы режут прядь в slo-mo (60fps → 24fps в постпродакшене).

#### Сцена 4 — Step 03 «Бритьё» (6–8s)

- **План:** макро. Опасная бритва скользит по щеке через тёплую пену.
- **Идея:** «Ритуал, забота, медленное удовольствие.»
- **Свет:** тёплый, контровой свет на пене и стали бритвы.

#### Сцена 5 — Step 04 «Финиш» (5–6s)

- **План:** общий или средний. Мастер распыляет аромат / делает финальную укладку. Камера ловит свет, проходящий через распылённый аромат (light beam).
- **Идея:** «Завершение, выход в новый облик.»

#### Сцены 6–8 — Interior loops (по 3–5s каждая)

- Деталь крана / раковины (вода капает в slow motion).
- Полка с косметикой (lateral pan).
- Кресло и инструменты в ожидании клиента.

### 5.4 Технические параметры съёмки

| Параметр              | Значение                                |
|-----------------------|------------------------------------------|
| Разрешение             | 4K UHD (3840×2160) минимум               |
| Frame rate             | 24fps (cinematic) или 60fps (для slo-mo) |
| Кодек на запись        | H.265 10-bit (если поддерживается камерой) или ProRes |
| ISO                    | < 1600 (минимизируем шум)                |
| Aperture               | f/2.8 – f/4 (тонкая глубина резкости)    |
| Картинка               | LOG / RAW (для последующего grading'а)  |
| Стабилизация           | Gimbal / Tripod / Slider (без рук)       |

### 5.5 Постпродакшен после пере-съёмки

То же, что в § 2.2 + § 3:
1. Импорт RAW/LOG.
2. Sync, отбор лучших дублей.
3. Цветокоррекция (раскрашиваем LOG, применяем custom LUT).
4. Резка под loop'ы.
5. Кадрирование под разные aspect-ratio.
6. Экспорт в WebM/MP4 + AVIF/WebP poster'ы.
7. Загрузка в Sanity.

### 5.6 Бюджет (план B, ориентировочно)

| Статья                        | Стоимость, ₽       |
|-------------------------------|--------------------|
| Оператор + камера + базовое освещение (1 смена) | 30 000 – 60 000 |
| Постпродакшен (монтаж + colorgrade) | 15 000 – 30 000 |
| Дополнительная аренда света (если нужна) | 5 000 – 15 000 |
| Запас на forced reshoot       | 10 000             |
| **Итого**                     | **60 000 – 115 000** |

Решение Айрата по плану B принимается после аудита существующего материала на 1-й неделе.

## 7. A11y и UX-нюансы видео

### 7.1 Безопасность для уязвимых пользователей

- **`prefers-reduced-motion: reduce`** — все автоплей-видео заменяются статичным poster'ом. См. компонент `<HeroVideo />` выше.
- **Vestibular-чувствительность** — никаких быстрых пан-движений / резких зумов / стробоскопов.
- **Эпилепсия** — никаких вспышек > 3 раз в секунду.

### 7.2 Скрин-ридеры

- Каждое `<video>` имеет `aria-label` с описанием контента (например, «Атмосфера салона BASE Premier» или «Шаг 2: Стрижка волос»).
- `<video>` БЕЗ controls (никаких кнопок play/pause), потому что видео декоративное.
- Для скрин-ридера контент видео — только `aria-label`. Никакого audio description не нужно (т.к. нет повествования).

### 7.3 Тумблер звука (header)

Хотя на сайте все видео по умолчанию muted, у нас есть глобальный тумблер «Звук» в шапке (см. DESIGN-BRIEF.md § 2.1).

**Когда тумблер ВКЛ:**
- Активируются sound design эффекты (см. BRAND.md § sound).
- НЕ означает, что видео начинают играть со звуком (аудио-дорожка вообще не записана).

**Когда тумблер ВЫКЛ:**
- Никаких звуков нигде.

### 7.4 Mobile data

На мобильных при подключении 2G/3G:
- Используем `navigator.connection.effectiveType` (если поддерживается).
- При `slow-2g` или `2g` — видео не загружаются, показывается только poster.
- При `3g` — загружается mp4 (более низкий размер на сжатый бит), webm пропускается.
- При `4g+` — стандартное поведение (webm).

```tsx
function getEffectiveConnection() {
  if (typeof navigator === 'undefined') return '4g'
  // @ts-ignore — connection не в типах
  const conn = navigator.connection
  return conn?.effectiveType ?? '4g'
}
```
