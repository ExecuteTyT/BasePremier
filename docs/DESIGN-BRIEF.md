## 2. ГЛОБАЛЬНЫЕ ЭЛЕМЕНТЫ

> Это компоненты, которые **присутствуют на каждой странице** и не пересоздаются при page transition.

### 2.1. Header (sticky, transparent → solid on scroll)

**Структура.**

```
┌──────────────────────────────────────────────────────────────────────┐
│  [BP]                                            [🔇]  [Записаться]  │
│        ↑                                          ↑          ↑       │
│   логотип-                                    тумблер    magnetic   │
│   монограмма                                   звука      CTA        │
└──────────────────────────────────────────────────────────────────────┘
```

**Поведение.**
- В состоянии «top» (`scrollY < 100`) — фон `transparent`, контур-only логотип в `--fg-primary`.
- В состоянии «scrolled» (`scrollY >= 100`) — фон `rgba(10, 10, 11, 0.85)` + `backdrop-blur(20px)` + `border-bottom: 1px solid var(--border)`. Переход 320ms `--ease-out-quart`.
- При прокрутке вверх (`scrollY уменьшается > 50px`) — header показывается, даже если был скрыт.
- При прокрутке вниз (после `scrollY > 600`) — header плавно скрывается (`translateY(-100%)`, 280ms).

**Компонент:** `<Header />` — клиентский, `'use client'`. Использует `useScrollDirection()` хук.

**Высота.** Desktop: 80 px. Mobile: 64 px.

**Mobile-меню.** При экране < 768 px — кнопка «Меню» вместо «Записаться» в правой части. Открывает full-screen overlay с навигацией, закрывается крестиком и swipe-down.

**Логотип «BP».** Линейная монограмма, выполненная в Fraunces Bold, opsz: 144, italic: 0, размер шрифта 32 px (desktop) / 24 px (mobile). При hover — лёгкое сжатие вертикальной оси через `font-variation-settings: 'wdth' 95`. При 5 кликах подряд → Easter egg: на 1.5 сек запускается фрагмент 3D-сцены прямо в логотипе (см. § 10).

**Тумблер звука.** Свернутая иконка ear (Lucide). Состояния:
- `off` (по умолчанию): иконка `volume-x` в `--fg-muted`.
- `on`: иконка `volume-2` в `--fg-primary`.
- При клике — переключение, состояние сохраняется в `localStorage` под ключом `bp:sound-enabled`.

**Magnetic CTA «Записаться».**
- Базовое состояние: `bg: --accent`, `color: --accent-fg`, padding 12 24, radius 4 px.
- При hover: `magnetic` эффект (см. `BRAND.md` § 8.3) — кнопка следует за курсором с радиусом 60 px.
- При клике — открывает YClients widget в модалке (см. § 2.5).

### 2.3. Custom cursor

**Структура.**

```
[ ⊙ ]   ← курсор по умолчанию: 8px кружок, --fg-primary, opacity 0.7
       └─ trail: 32px кружок, border 1px --border, opacity 0.4, lerp delay
```

**Состояния.**

| Состояние | Триггер | Внешний вид |
|---|---|---|
| `default` | по умолчанию | 8 px кружок `--fg-primary`, trail 32 px border |
| `hover-link` | `:hover` на `<a>` или `[role=button]` | trail увеличивается до 56 px, пропадает основной кружок |
| `hover-magnetic` | `:hover` на CTA с классом `magnetic` | trail становится `bg: --accent`, текст внутри trail `var(--accent-fg)` |
| `hover-image` | `:hover` на `<Image data-cursor="view">` | в trail появляется текст «View» |
| `text-input` | `:focus` на `<input>`, `<textarea>` | курсор превращается в стандартный I-beam (отключаем custom) |
| `disabled` | `prefers-reduced-motion: reduce` | custom cursor выключается полностью |

**Производительность.**
- Movement через `requestAnimationFrame` с `transform: translate3d`. Не использовать `top`/`left`.
- Lerp factor 0.15 для trail.
- Mobile (`hover: none`) — не рендерим компонент вообще (`tree-shaken` через `useIsTouchDevice()` хук).

**Компонент.** `<CustomCursor />` — клиентский. Внутри `<App>`, рендерится один раз. Прячется на `:focus` для `<input>`.

### 2.4. Page transitions

> Используем Framer Motion + `next/navigation` `useSelectedLayoutSegment`.

**Сценарий.**

1. Клик по `<Link>` → старая страница `exit`-анимация:
   - `opacity: 1 → 0`
   - `scale: 1 → 0.98`
   - `duration: 320ms`, `ease: --ease-in-out-cubic`
2. Между exit и enter — пустой экран на 80 ms, в центре — короткая анимация BP-монограммы (рисуется stroke от 0 до 100%). Если route загружается быстро (≤ 80ms) — этот блок пропускается.
3. Enter:
   - `opacity: 0 → 1`
   - `scale: 0.98 → 1`
   - `y: 16px → 0`
   - `duration: 480ms`, `ease: --ease-out-expo`

**Исключения.**
- Скролл при page transition сбрасывается в `0`.
- На `/quiz` и `/bp` — отдельная анимация (см. соотв. секции).
- При `prefers-reduced-motion: reduce` — instant cross-fade без scale/translate.

**Компонент.** `<PageTransition>` — обёртка вокруг `{children}` в `app/layout.tsx`.

### 2.5. YClients booking modal

**Триггер.** Любая кнопка с классом `[data-booking]` → открывает модалку с виджетом YClients.

**Модалка.**
- Full-screen на mobile, центрированная 720×680 на desktop.
- Backdrop: `rgba(10, 10, 11, 0.85)` + `backdrop-blur(8px)`.
- Closing: ESC, клик по backdrop, кнопка ×.
- Inside: `<YClientsWidget company_id={…} mode={…} />` (см. `ARCHITECTURE.md` § 9.4).

**Префикс через UTM.**
- Если триггер — кнопка на странице мастера, `?staff_id={id}` передаётся в виджет.
- Если триггер — кнопка на странице услуги, `?service_id={id}` передаётся.
- Если триггер — кнопка результата квиза, оба передаются.

### 2.6. Loader (initial page load)

**Только на первом заходе на сайт** (sessionStorage check). При повторных переходах — не показываем.

**Сценарий.**

1. Чёрный экран `--bg-primary`.
2. В центре — BP-монограмма, рисуется stroke в течение 1.5 сек (`stroke-dashoffset` от full к 0).
3. После full-stroke — небольшая пауза 200ms, затем монограмма «рассыпается» (опять-таки через stroke-dashoffset, но в другую сторону).
4. На месте монограммы появляется главная страница (cross-fade, 480ms).

**Минимальная длительность.** 1.5 сек, даже если контент загружен раньше. Это часть впечатления.

**Skip.** При `prefers-reduced-motion: reduce` или повторных заходах — loader пропускается.

**Альтернатива.** Если LCP < 1.0 сек — loader конкурирует с реальным контентом. Поэтому: на повторных заходах — пропускаем; в первый заход — даём loader как часть впечатления.

### 2.8. Sound system

**Звуки в проекте** (все из Freesound, Zapsplat, Pixabay; CC0 / CC-BY 4.0):

| Событие | Длина | Объём |
|---|---|---|
| Hover на CTA | 80–120 ms | < 8 KB |
| Click на CTA | 100–150 ms | < 10 KB |
| Page transition | 600 ms | < 30 KB (выбираем тонкий sweep) |
| Подтверждение записи (после YClients success) | 400 ms | < 20 KB |
| Easter-egg active (5 кликов лого) | 1500 ms | < 40 KB |

**Компонент.** `<SoundProvider>` в `app/layout.tsx`. Использует `Howler.js` (~7 KB gzip).

**Анимации.**
1. **На загрузке** (после loader, см. § 2.6):
   - H1 — text reveal через split (каждый символ из `y: 100% → 0`, stagger 30ms, общая длительность 800ms, `--ease-out-expo`).
   - Эпитеты — text reveal через split-words (каждое слово, stagger 60ms).
   - Sub — простой `opacity: 0 → 1`, `y: 16 → 0`, delay 400ms.
   - CTA — magnetic-fade-in с задержкой 800ms.
   - 3D-BP — появляется через scale `0 → 1`, 900ms, `--ease-out-expo`, delay 600ms.
2. **При hover на 3D-BP** — буква поворачивается следом за курсором (mouse-tracking).
3. **При scroll** — H1 и H2 слегка уменьшаются (`scale 1 → 0.92`) и тускнеют (`opacity 1 → 0.4`), 3D-BP уменьшается аналогично.

**Hero-видео.**
- WebM VP9 + MP4 H.264 fallback (Safari).
- Mobile-версия: 720×1280, ≤ 1.5 MB.
- Desktop-версия: 1920×1080, ≤ 3 MB.
- Все детали — в `VIDEO-STORYBOARD.md`.
- Если `prefers-reduced-motion: reduce` — вместо видео poster-кадр.
- Если `navigator.connection.saveData === true` — вместо видео poster-кадр.

**3D-BP монограмма.**
- Three.js, lazy-loaded только когда секция в viewport.
- Геометрия: extruded SVG-path буквы B и P (через `THREE.ExtrudeGeometry` с `depth: 12, bevelEnabled: true, bevelThickness: 1.2`).
- Material: `MeshPhysicalMaterial` с `roughness: 0.3, metalness: 0.6, color: 0xF5F5F2, clearcoat: 0.4`.
- Освещение: 1 directional light (key) с правым углом 45°, 1 ambient (fill) низкой интенсивности.
- Camera: PerspectiveCamera, 35° FOV, position `[0, 0, 25]`.
- Mouse-tracking через `useFrame` хук — буква вращается на ±15° по yaw и pitch в зависимости от позиции курсора.
- При `prefers-reduced-motion: reduce` — статичная (без вращения).
- При mobile — вращение только по yaw, pitch фиксирован.
- Performance budget: 60 fps на mid-range device (Galaxy A54), не больше 5% CPU usage.
- Fallback (если Three.js падает) — статичная SVG.

**CTA блок.**
- Primary: «Записаться» (magnetic, открывает YClients modal).
- Secondary: «Услуги» (текст-link с underline-on-hover, ведёт на /services).
- Под CTA: маленький текст «Час времени · с 10:00 до 21:00» (Inter Mono 12px, `--fg-muted`).

**«Скрол» индикатор.**
- Вертикальная линия 1×40 px, `--fg-muted`.
- Под ней — текст «scroll» вертикально (rotated 90°), Inter Mono 10px, `--fg-muted`.
- Анимация: линия растёт от 0 до 40 px и обратно, петля 2 сек.

## 8. /QUIZ

**Цель.** Конверсионный игровой формат. По итогу — рекомендация мастера и услуги, кнопка «Записаться» с предзаполнением.


**Особенности.**

- Прогрессбар в верхней части.
- Кнопка «Назад» отключена на первом шаге.
- На последнем шаге — кнопка «Завершить».

**Анимации.**
- Смена шагов: текущий exit (`x: 0 → -100%`, opacity), новый enter (`x: 100% → 0`).

**Logic.** Mapping ответов → мастера и услуги:
- `faceShape: oval, hairLength: short` → `master: марат, service: муж. стрижка`
- `faceShape: round, hairLength: long` → `master: вячеслав, service: муж. стрижка с бородой`
- ... и т.д. (полная таблица — в `quiz-mapping.json`, генерится в фазе 4)

## 10. /BP — 3D-МОНОГРАММА

**Цель.** Easter-egg + PR-фишка. Полноэкранная 3D-сцена с буквой BP.

**Layout.**

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│                                                                     │
│                  ╔══════════╗                                       │
│                  ║          ║                                       │

│                  ╚══════════╝                                       │

│                                                                     │
│         [← На главную]                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**3D-сцена.**
- Three.js + R3F.
- Геометрия: высокополигональная буква BP (extruded SVG path с beveled edges, ~5000 polygons).
- Material: physical с metalness 0.7, roughness 0.2, clearcoat 0.5. Цвет — `--fg-primary` (#F5F5F2).
- Освещение: 3 directional lights (key, fill, rim) + HDR environment (можно скачать бесплатный из Polyhaven).
- Camera: orbital control (drei `<OrbitControls />`), enable damping. Limits: zoom min 5, max 50; vertical angle 0-180°.
- Мышка: drag для вращения, скрол для zoom. Touch — двупальцевый pinch + drag.

**Поведение.**
- Автозапуск: камера медленно облетает букву по кругу (1 оборот за 30 сек), пока пользователь не начнёт взаимодействовать.
- Когда пользователь начинает drag — autoplay останавливается.

**Performance.** Lazy-loaded, первая загрузка может быть до 1.5 сек (но это OK для Easter-egg). Polyhaven HDR — ≤ 200 KB.

**Доступ.** Прямой URL `/bp`. Также — после 5 кликов на лого в header (см. § 2.1) — пользователь перенаправляется сюда.

## 13. A11Y — ОБЩИЙ ЧЕКЛИСТ

> Применяется ко всем страницам. Тестируется через axe-core (см. `.claude/agents/a11y-auditor.md`).

### 13.1. WCAG 2.2 AA — обязательный минимум

- [x] Все текст ≥ contrast ratio 4.5:1 на background.
- [x] Все интерактивные элементы ≥ contrast ratio 3:1 в обоих состояниях.
- [x] Все изображения с информационным значением имеют `alt`.
- [x] Декоративные изображения — `alt=""` или `role="presentation"`.
- [x] Все формы — labelled (через `<label>` или `aria-label`).
- [x] Все кнопки — реальные `<button>`, не `<div onClick>`.
- [x] Все ссылки — реальные `<a href>`, не `<div onClick>`.
- [x] Tab-order логичный.
- [x] `:focus-visible` явно отображается на всех интерактивных элементах.
- [x] Skip-to-content ссылка работает.
- [x] `prefers-reduced-motion: reduce` отключает все анимации (не только декоративные).
- [x] Все видео — без автозвука; либо mute по умолчанию + пользовательский toggle.
- [x] Все картинки — с lazy-loading кроме hero (priority).
- [x] Заголовки идут в правильной иерархии (`h1` → `h2` → `h3`, без перепрыгивания).
- [x] На каждой странице ровно один `<h1>`.

### 13.2. Spec для интерактивных компонентов

| Компонент | Поведение клавиатуры |
|---|---|
| Custom cursor | Не интерферирует с focus-management |
| Carousel (отзывы) | Стрелки `←/→` или `Tab` для перехода |
| Quiz | `Tab` между опциями, `Enter` для выбора |
| Accordion (FAQ) | `Enter` или `Space` для открытия/закрытия |
| Magnetic CTA | `Enter` срабатывает как клик; magnetic выключен на focus |
| Sound toggle | `Space` переключает; aria-pressed обновляется |
| Page transitions | `prefers-reduced-motion` → instant |
