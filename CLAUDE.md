## 2. БРЕНД И TONE OF VOICE

### 2.1. Три эпитета (вшиваются во все тексты)

> **Дорого. Премиально. С собственным шармом.**

### 2.2. УТП (на hero и в meta-description)

> «Уникальный интерьер, профессиональные барберы, высокие стандарты — всё для того, чтобы вы чувствовали себя на высоте.»

Можно перефразировать в 3 варианта (см. CONTENT-BRIEF.md), но эти три якоря — **интерьер · мастера · стандарты** — сохраняются.

### 2.3. Целевая аудитория

- **Ядро 80%:** мужчины 30–40 лет, владельцы бизнеса / высокооплачиваемые специалисты, доход 300+ тыс ₽/мес. Идут за обстановкой, ощущениями и сервисом.
- Конкурент в их голове: **TopGun**.

### 2.4. Tone of voice — правила копирайта

Эти правила обязательны во всех текстах сайта:

| Правило               | Как                                                                                                                                                      |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Обращение**         | На «вы». Никогда «ты». В сомнительных случаях — переформулировать безлично.                                                                              |
| **Стиль**             | Сдержанный и серьёзный. Эталоны — Tom Ford, Aesop. **Без юмора и без иронии.**                                                                           |
| **Длина**             | Короткие (3–5 слов) для CTA и заголовков. Средние (1–2 предложения) для описаний. **Никаких параграфов 5+ строк.**                                       |
| **Иностранные слова** | Микс. Устоявшиеся барберские термины (fade, taper, дренаж, контуринг) — оставляем. Маркетинговые англицизмы (premium, exclusive, lifestyle) — переводим. |
| **Кавычки**           | «Ёлочки», не "программистские". Тире — длинное (—).                                                                                                      |
| **Цифры**             | Цены — без копеек: `2 700 ₽`, не `2700 руб` и не `2700₽`. Неразрывный пробел перед ₽.                                                                    |
| **Время**             | `1 ч` или `1 ч 30 мин`. Не `1:30` и не `90 минут`.                                                                                                       |

### 2.5. Что бренд НЕ хочет говорить о себе

В копирайте, дизайне и фото **избегать** ассоциаций с:

- Мужским клубом с виски и сигарами (хотя клиенты — мужчины 30–40)
- Молодёжным hipster-локалом
- Семейной парикмахерской
- Гламурным салоном красоты
- Дешёвым массмаркетом с премиум-фасадом

Если в брифе/копии появляется хоть один такой намёк — переписать.

### 2.6. Слоган

Текущий: «Стиль, атмосфера, мастерство — больше, чем просто барбершоп». **Допускается доработка** в CONTENT-BRIEF.md → раздел «Манифест».

## 3. ДИЗАЙН-ИНВАРИАНТЫ

Полный гайд → `docs/BRAND.md`. Здесь только то, что **не меняется** ни при каких условиях.

### 3.1. Цвета

```
--bg-primary:   #0A0A0B  (почти чёрный, не #000000)
--bg-secondary: #131316  (для cards/sections)
--fg-primary:   #F5F5F2  (off-white, не #FFFFFF)
--fg-muted:    #9A9A98
--accent:      #1B2A4E   (Navy — единственный акцент)
--accent-fg:   #F5F5F2
--border:      rgba(245,245,242,0.08)
--success:     #6F8B6F   (приглушённый, не bootstrap-green)
--error:       #B95A4F   (приглушённый, не bootstrap-red)
```

**Запрет:** никакого `#FFFFFF` и `#000000` в коде. Только токены через CSS variables и Tailwind theme.

### 3.2. Тема — только dark

Светлой темы **нет**. Не делаем переключатель. Не добавляем `prefers-color-scheme`. Сайт всегда тёмный — это часть брендинга.

### 3.3. Шрифты (бесплатные Google Fonts)

| Назначение          | Шрифт                               | Где                |
| ------------------- | ----------------------------------- | ------------------ |
| **Display / H1-H2** | Fraunces (variable, opt-size 9-144) | `next/font/google` |
| **Body / UI**       | Inter (variable)                    | `next/font/google` |
| **Numerals / mono** | JetBrains Mono (variable)           | для цен и кодов    |

Загрузка только через `next/font` — никаких `<link>` в `<head>`. Self-hosted, no CLS.

Кириллица: subset `["latin", "cyrillic"]` для всех трёх. **Допускается:** H1 на английском + перевод подзаголовком на русском (как у Aesop в РФ).

### 3.4. Логотип = монограмма «BP»

> **ВАЖНО:** наш якорь — это буквы **BP** (от BASE Premier), **не «X»** (которое было в исходном ресерче — это устаревшее).

Везде где упоминается «3D-X» в любых документах — читай «3D-BP».

### 3.5. Spacing scale

8-pt grid: `4 8 12 16 24 32 48 64 96 128 160 192 256` (px). В Tailwind v4 — через `--spacing-*` токены.

### 3.6. Animation easings (одни и те же на весь проект)

```css
--ease-out-expo: cubic-bezier(0.19, 1, 0.22, 1); /* основной */
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1); /* для UI */
--ease-in-out-cubic: cubic-bezier(0.65, 0, 0.35, 1); /* для page transitions */
```

**Запрет:** `ease`, `ease-in`, `ease-out`, `linear` — это дефолты браузера, выглядят дёшево. Только токены.

### 3.7. Уровень анимаций — максимум

Делаем всё: видео-фон, кинетика, parallax, кастомный курсор, page transitions, marquee, magnetic buttons, drag-reveal, horizontal scroll. Цель — Awwwards Site of the Day.

**Но** все wow-эффекты обязаны:

- Уважать `prefers-reduced-motion: reduce` (отключаются полностью)
- Не блокировать LCP > 2.5s
- Не уменьшать INP < 200ms
- Не нарушать a11y (focus-visible работает поверх всего)

### 3.8. Sound design

Тихие звуки на hover/transition/успех записи. **По умолчанию OFF**, тумблер 🔇 в углу шапки. Источник — Freesound / Zapsplat / Pixabay (CC0 / CC-BY-4.0). Lazy-loaded.

## 5. WORKFLOW ДЛЯ CLAUDE CODE

### 5.1. Принципы

1. **Один тикет = один Pull Request = один git worktree.** Никогда не работаем в нескольких темах в одной ветке.
2. **TDD-light:** для бизнес-логики (YClients API, форматирование цены, time slots) — Vitest. Для UI — Playwright e2e на критичных путях (открыть hero → кликнуть «Записаться» → виджет YClients).
3. **Storybook не делаем.** Дешевле и качественнее — `app/dev/` маршрут с галереей компонентов, доступный только в `NODE_ENV=development`. ⚠️ Папки с префиксом `_` (например `_dev`) в Next.js App Router не создают маршруты — Next.js считает их приватными. Используем `app/dev/` без подчёркивания.
4. **Перед коммитом — всегда:** `pnpm typecheck && pnpm lint && pnpm test`. Это в hooks (см. `.claude/settings.json`).
5. **Новый компонент** — всегда сначала ищем в `src/components/ui/` существующий. Если нет — обсуждаем с собой (через `claude --print`) ДО написания кода.

## 6. ПРАВИЛА КОДА

### 6.1. TypeScript — strict, без `any`

`tsconfig.json` имеет `"strict": true` и `"noUncheckedIndexedAccess": true`. Если возникает желание написать `any` — пишем `unknown` и сужаем через `zod`.

### 6.2. Серверные/клиентские компоненты

- **По умолчанию** — Server Component (RSC).
- `'use client'` ставим **только** если компонент использует hooks (`useState`, `useEffect`, `useScroll`), event handlers (`onClick`), или браузерные API (`window`, `document`, `IntersectionObserver`).
- Все секции главной — клиентские (нужны GSAP/Framer).
- Footer, Header (без интерактива), JSON-LD — серверные.

### 6.3. Tailwind v4 — CSS-first

Все токены живут в `src/styles/tokens.css` под директивой `@theme`. **Не кладём** магические числа в JSX (`px-[17px]`). Если нужно что-то нестандартное — добавляем токен.

### 6.4. Именование

```
PascalCase     — компоненты React (HeroSection.tsx)
camelCase      — функции, переменные
kebab-case     — папки и URL-роуты (/barbers/marat, src/components/hero-section)
SCREAMING      — env vars (YCLIENTS_API_KEY)
```

### 6.5. Импорты — алиасы

```
@/components/* → src/components/*
@/lib/*        → src/lib/*
@/styles/*     → src/styles/*
@/types/*      → src/types/*
@/sanity/*     → src/sanity/*
```

Никаких `../../../components`. Если возникло — рефактор алиасов.

### 6.6. Цены и числа

Используем единственную утилиту `formatPrice()` из `@/lib/format`:

```ts
formatPrice(2700); // "2 700 ₽"
formatPriceRange(1800, 2700); // "1 800 – 2 700 ₽"
formatDuration(90); // "1 ч 30 мин"
```

**Никогда** не форматируем вручную в JSX.

### 6.7. SEO-инвариант: каждая страница ОБЯЗАНА иметь

- `metadata.title` уникальный (≤ 60 символов)
- `metadata.description` уникальный (140-160 символов)
- `metadata.openGraph` — title, description, image (1200×630), url
- `metadata.alternates.canonical` — абсолютный URL
- JSON-LD соответствующего типа (`HairSalon`, `Service`, `Person`, `Article`, `BreadcrumbList`)

Без этих 5 штук — `pnpm build` должен упасть. (Делается через `seo-auditor` агента в pre-push hook.)

## 7. КРИТИЧЕСКИЕ ИНВАРИАНТЫ (НЕ НАРУШАТЬ)

### 7.1. Что нельзя сломать при миграции

1. **Записи в YClients** — миграция на новый сайт не должна повлиять на существующие записи. Виджет YClients подключается к тому же `company_id` и `form_id`, что и сейчас.
2. **Старые URL** — нужны 301-редиректы для всех значимых страниц старого сайта. Список — в `docs/ARCHITECTURE.md` § Redirects. Конфиг в Nginx.
3. **SEO-позиции** — несмотря на текущую слабую позицию (8-9 страница), не теряем имеющийся ссылочный вес. Делаем sitemap.xml, robots.txt, Я.Вебмастер, переадресации старых URL.
4. **Программа лояльности** — не трогаем, она в YClients. На сайте — только упоминание факта.

### 7.2. Что нельзя на сайте (юридически и по бренду)

- ❌ Никаких упоминаний алкоголя (виски и т.д.) — заказчик подтвердил, нет такого
- ❌ Никаких "До/После" — заказчик отказался от блока
- ❌ Никаких упоминаний Павлюхины — этот филиал продан
- ❌ Никакого юмора, иронии, мемов
- ❌ Никаких лайфстайл-байтов в духе «джентльмен, ритуал, истинный мужчина» (это TopGun, наш конкурент — мы не повторяем)
- ❌ Никаких сертификатов онлайн (физические в салоне)

### 7.3. Что обязательно на сайте

- ✅ Cookie-banner (152-ФЗ + GA-совместимая практика)
- ✅ Согласие на обработку ПД при отправке любой формы
- ✅ Корректные реквизиты ИП в футере
- ✅ JSON-LD `HairSalon` с реальными данными (адрес, часы, телефон, рейтинг)
- ✅ Я.Метрика + Я.Вебмастер
- ✅ Sitemap.xml + robots.txt

### 7.4. Что не собираем через формы

Заказчик решил: **никаких форм, собирающих ПД** на сайте. Запись идёт через виджет YClients (там сами разбираются с 152-ФЗ). У нас на сайте только:

- Кнопка «Записаться» → открывает YClients widget
- Кнопка «Telegram / WhatsApp» → открывает мессенджер
- Форма «Узнать акции» — НЕ ДЕЛАЕМ
- Форма «Обратная связь» — НЕ ДЕЛАЕМ
- Форма квиза — без отправки на бэк, просто показывает рекомендацию + кнопку «Записаться» (которая откроет YClients с предзаполненной услугой через UTM)

Это снимает с нас обязанность регистрироваться в РКН как оператор ПД.
